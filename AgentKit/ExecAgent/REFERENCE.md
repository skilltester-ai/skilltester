# ExecAgent Reference

## Active Execution Bundle

ExecAgent must be launched in two separate code terminals.

1. one terminal for `baseline`
2. one terminal for `with_skill`

SafeTest is no longer part of ExecAgent.
Security execution now belongs to `SpecAgent` and must follow:

- `AgentKit/SpecAgent/workflow.md`
- `AgentKit/SpecAgent/prompt.md`
- `AgentKit/SpecAgent/SpecLibrary/SafeTest/`

The two stage terminals may be launched independently or in parallel.
Inside each individual stage, tasks must still execute one by one.
Parallel stages are allowed. What remains forbidden is task-level concurrency
inside the current stage terminal.

The only concrete runtime entrypoints are:

- `AgentKit/ExecAgent/baseline/prompt.md`
- `AgentKit/ExecAgent/baseline/workflow.md`
- `AgentKit/ExecAgent/withskill/prompt.md`
- `AgentKit/ExecAgent/withskill/workflow.md`

The canonical bundle for each functional task is:

- `results/`
- `task_metrics.json`
- `worklog.log`
- `start_timestamp.json`
- `end_timestamp.json`

## JSON Skeleton Rule

These JSON files are first created as skeletons by scripts and then backfilled by the agent with real fields:

- `AgentKit/ExecAgent/utils/generate_JSON/generate_task_metrics.py`
- `AgentKit/ExecAgent/utils/generate_JSON/generate_stage_metrics.py`
- `AgentKit/ExecAgent/utils/calculate_timestamp_diff.py`

Both `start_timestamp.json` and `end_timestamp.json` are generated by:

- `AgentKit/ExecAgent/utils/write_system_timestamp.py`

Do not hand-write timestamp JSON or create it through `echo`, `date`, `touch`, or any other method.
Canonical `start_timestamp.json` / `end_timestamp.json` must be the JSON emitted by that script, not plain-text timestamp strings.

## Stage Outputs

Each stage must preserve:

- `results/baseline/metrics.json`
- `results/with_skill/metrics.json`
- `results/{skill_name}/exec/results/agent_worklog.log`

`baseline` and `with_skill` write to separate stage directories and may advance
independently. A baseline terminal must not write with-skill artifacts, and a
with-skill terminal must not write baseline artifacts.

The responsibility of stage-level `metrics.json` is to aggregate all `task_metrics.json` files in that stage, not to replace task-level raw evidence.

## Time Evidence

The first action of each stage must write:

- `results/<mode>/stage_start_timestamp.json`

And it must be generated by `AgentKit/ExecAgent/utils/write_system_timestamp.py`.

Before reading `task_description.md`, each task must first write:

- `results/<mode>/tasks/{task_id}/start_timestamp.json`

The last step of each task must write:

- `results/<mode>/tasks/{task_id}/end_timestamp.json`

Then `calculate_timestamp_diff.py` must be called, and the following fields must be backfilled:

- `time`
- `total_time_seconds`
- `time_estimate_basis`

The only valid source of `time` / `total_time_seconds` is the output of `calculate_timestamp_diff.py`.
Do not recompute task time from `stage_start_timestamp.json`, the previous task's `end_timestamp.json`, `timer.log`, default values, or any inference method.

## Required Task Metrics Fields

`task_metrics.json` should preserve at least:

- `task_id`
- `mode`
- `status`
- `task_start_timestamp`
- `time`
- `total_characters`
- `estimated_total_tokens`
- `token_estimate_basis`

`with_skill` tasks should additionally record:

- `skill_invocation_attempted`
- `skill_invocation_success`

## Logging

- Global log: `results/{skill_name}/exec/results/agent_worklog.log`
- Single-task log: `results/<...>/worklog.log`

Format references:

- `AgentKit/ExecAgent/schema/agent_worklog.txt`
- `AgentKit/ExecAgent/schema/task_worklog.txt`

## Aggregation Rule

- `results/<mode>/metrics.json` must be generated by aggregating all `task_metrics.json` files of the same mode
- `Tasks.json` is not an ExecAgent artifact; it is generated by SpecAgent in the next stage based on this evidence
- execution artifacts for security probes have already been moved to `results/{skill_name}/spec/results/security/`

## Non-Canonical Legacy Outputs

The following files are no longer canonical required artifacts in the current ExecAgent flow:

- standalone per-task summary sidecars outside the canonical task bundle
- ad hoc task-local metrics clones beyond `task_metrics.json`
- ad hoc token / execution log sidecars outside the canonical worklog and metrics chain
- `artifacts/`
- `scripts/` as a fixed directory
- `benchmark_metrics.json`
