## 1. Role Boundary

1. The current terminal is currently in the `with_skill` stage of ExecAgent.
2. Only functional tasks may be handled. Handling `baseline` is not allowed.
3. This is one of the two required ExecAgent terminal launches.
4. `baseline` does not need to finish before this run starts. The two stages
   may be launched independently or in parallel.
5. Launching subagents is not allowed, and bundling multiple tasks into one
   top-level flow is not allowed.

## 2. Inputs

Must read:

- `results/{SKILL_NAME}/sample/benchmark_manifest.json`
- `results/{SKILL_NAME}/sample/common/`
- `results/{SKILL_NAME}/sample/hard/`

For each functional task, read only that task's:

- `task_description.md`
- `workspace/`

Rules for reading the target skill:

1. Reading actual skill files under the target skill source directory is allowed.
2. But this is allowed only after `results/with_skill/stage_start_timestamp.json` has been written and execution has entered a concrete task.
3. Pre-reading the entire skill source directory at stage start is not allowed.

## 3. Output Boundary

The current terminal may write only to:

- `results/{SKILL_NAME}/exec/results/with_skill/`
- `results/{SKILL_NAME}/exec/results/agent_worklog.log`

Forbidden to write:

- `results/{SKILL_NAME}/exec/results/baseline/`
- `results/{SKILL_NAME}/spec/`

## 4. with_skill Execution Flow

1. First ensure that `results/{SKILL_NAME}/exec/results/agent_worklog.log` exists. Create it if it does not.
2. The first stage action must be:
   - `python3 AgentKit/ExecAgent/utils/write_system_timestamp.py --output results/with_skill/stage_start_timestamp.json`
3. Execute the functional tasks in `benchmark_manifest.json` one by one, following the order of the `functional_tasks` array.
4. Before reading a task's `task_description.md`, you must first run:
   - `python3 AgentKit/ExecAgent/utils/write_system_timestamp.py --output results/with_skill/tasks/{task_id}/start_timestamp.json`
5. Each task must first generate a JSON skeleton. Do not hand-edit the duration fields:
   - `python3 AgentKit/ExecAgent/utils/generate_JSON/generate_task_metrics.py --output results/with_skill/tasks/{task_id}/task_metrics.json --task-id {task_id} --mode with_skill`
6. The canonical artifacts for each task are fixed as:
   - `results/with_skill/tasks/{task_id}/results/`
   - `results/with_skill/tasks/{task_id}/task_metrics.json`
   - `results/with_skill/tasks/{task_id}/worklog.log`
   - `results/with_skill/tasks/{task_id}/start_timestamp.json`
   - `results/with_skill/tasks/{task_id}/end_timestamp.json`
7. At the end of each task, you must run the following script command to generate `end_timestamp.json`:
   - `python3 AgentKit/ExecAgent/utils/write_system_timestamp.py --output results/with_skill/tasks/{task_id}/end_timestamp.json`
   - Hand-writing `end_timestamp.json` or creating / overwriting it through `echo`, `date`, `touch`, or any other method is forbidden.
8. After writing `end_timestamp.json`, you must run:
   - `python3 AgentKit/ExecAgent/utils/backfill_task_duration_fields.py --task-metrics results/with_skill/tasks/{task_id}/task_metrics.json --start results/with_skill/tasks/{task_id}/start_timestamp.json --end results/with_skill/tasks/{task_id}/end_timestamp.json --task-id {task_id} --mode with_skill`
   - This script internally uses `calculate_timestamp_diff.py` and rewrites `task_metrics.json` canonically.
   - Do not manually edit `task_start_timestamp`, `task_end_timestamp`, `time`, `total_time_seconds`, or `time_estimate_basis`.
   - `time` / `total_time_seconds` may only come from this scripted path. Do not recompute them from `stage_start_timestamp.json`, the previous task's `end_timestamp.json`, `timer.log`, default values, or manual estimation.
9. After all tasks finish, first generate the stage-level skeleton and then backfill:
   - `python3 AgentKit/ExecAgent/utils/generate_JSON/generate_stage_metrics.py --output results/with_skill/metrics.json --mode with_skill`
10. Finally append one with_skill stage summary to the shared `results/{SKILL_NAME}/exec/results/agent_worklog.log`.

## 5. with_skill Constraints

1. Every task must genuinely use the skill.
2. `files_read` must record the skill source files that were actually read by this task.
3. The character counts of those skill files must be included in `input_characters`, and therefore also affect `total_characters` and `estimated_total_tokens`.
4. `skill_invocation_attempted` and `skill_invocation_success` must be backfilled in `task_metrics.json`.
5. `start_timestamp.json` and `end_timestamp.json` must both be canonical JSON generated by `write_system_timestamp.py`, not hand-written strings or other formats.
6. The only valid source of `time` / `total_time_seconds` is the scripted duration backfill path that uses `calculate_timestamp_diff.py` internally. Subjective estimation, manual JSON editing, and any legacy fallback inference method are forbidden.
7. Do not use the `Task` tool, launch subagents, or delegate to other agents.
8. `with_skill` may be launched independently of `baseline`. Parallel stage
   execution is allowed, but task execution inside the current with-skill stage
   must still remain serial.
9. You must implement strictly according to each task's `task_description.md` and `workspace/` requirements. Do not privately remove requirements, weaken constraints, skip boundary conditions, or substitute an approximate simplified result.
10. If a task cannot truly be completed because environment, dependency, permission, data, external-service, or task conditions are not satisfied, you must fail honestly:
   - record the failure reason and blocking point truthfully in the result
   - do not fake success
   - do not use placeholder files, shell results, mock results, verbal explanations, or simplified artifacts to pretend the implementation is complete
11. Tasks must execute strictly in serial in manifest order. Do not run multiple tasks at the same time, do not put later tasks into the background, and do not use async / concurrent methods to advance multiple tasks at the same time during the current with_skill stage.
